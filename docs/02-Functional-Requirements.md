# Functional Requirements

---

# 1. Introduction

This document defines the functional requirements of the Smart Home IoT Platform.

The purpose of this document is to describe the expected functionality of the platform from a user's perspective. It serves as a reference for system design, implementation, testing, and future enhancements.

The requirements documented here represent both the current capabilities of the platform and the planned functionality that will be implemented in future development sprints.

---

# 2. System Actors

The platform currently supports the following primary actor.

## Administrator

The administrator is responsible for:

- Registering and managing IoT devices
- Monitoring device status
- Viewing dashboard statistics
- Monitoring application health
- Managing project investments
- Viewing activity history
- Managing system configuration
- Managing future automation rules

Future versions of the platform may introduce additional roles such as standard users and guest users.

---

# 3. Functional Modules

The Smart Home IoT Platform is divided into the following functional modules:

- Dashboard
- Device Management
- Activity Management
- Health Monitoring

- Automation *(Planned)*
- Analytics & Reports *(Planned)*
- Notifications *(Planned)*

# 4. Dashboard Module

## Description

The Dashboard provides a centralized overview of the Smart Home IoT Platform, allowing administrators to quickly monitor the overall system status.

## Functional Requirements

The system shall:

- Display the total number of registered devices.
- Display online devices.
- Display offline devices.
- Display active motion sensors.
- Display recent activities.
- Display backend health information.
- Display total project investment.
- Refresh dashboard data periodically.

**Status:** Implemented

---

# 5. Device Management Module

## Description

The Device Management module is responsible for registering, managing, updating, deleting, and monitoring all IoT devices connected to the platform.

## Functional Requirements

The system shall:

- Register new devices.
- Update existing devices.
- Delete registered devices.
- Display device information.
- Search devices.
- Filter devices.
- Monitor device status.
- Process heartbeat updates from IoT devices.
- Track device connectivity.

**Status:** In Progress *(Sprint 3)*

---

# 6. Activity Management Module

## Description

The Activity Management module records and displays events generated by connected IoT devices.

## Functional Requirements

The system shall:

- Record motion detection events.
- Record device activity.
- Display recent activities.
- Support activity filtering.
- Maintain historical activity logs.

**Status:** Implemented

---

# 7. Health Monitoring Module

## Description

The Health Monitoring module provides operational metrics for monitoring the backend application.

# 8. Project Investment Management Module

The Project Investment Management module tracks all hardware, infrastructure, networking, and software investments made while building the Smart Home IoT Platform.

## Functional Requirements

- Update investment records.
- Delete investment records.
- View investment history.
- Search investments.
- Filter investments by category.
- Display total project investment.
- Display investment analytics.
- Categorize investments.
- Track vendors and purchase dates.

# 14. Non-Functional Requirements

In addition to functional requirements, the platform shall satisfy the following quality attributes.

## Performance

- Provide fast API response times.
- Support concurrent client requests.
- Minimize database query latency.

## Scalability

- Support additional IoT devices.
- Support future feature expansion.
- Allow modular application growth.

## Security

- Secure communication using HTTPS.
- Validate incoming requests.
- Protect REST APIs.
- Support JWT authentication.

## Reliability

- Handle application errors gracefully.
- Maintain device connectivity.
- Ensure high service availability.

## Maintainability

- Follow layered architecture.
- Follow clean coding principles.
- Maintain comprehensive documentation.
- Support modular development.

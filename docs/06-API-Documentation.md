# API Documentation

---

# 1. Introduction

The Smart Home IoT Platform exposes RESTful APIs that enable communication between the frontend dashboard, IoT devices, and backend services.

All APIs follow REST principles and exchange data using JSON.

The API layer is designed to provide a consistent, secure, and scalable interface for managing devices, monitoring activities, collecting sensor data, and retrieving system health information.

---

# 2. API Standards

The backend follows the following API conventions.

## Protocol

- HTTPS

## Data Format

- JSON

## Character Encoding

- UTF-8

## Base URL

```
https://api.smarthome.sureshragam.in
```

For local development:

```
http://localhost:8080
```

---

# 3. Dashboard APIs

## Get Dashboard Summary

### Endpoint

```
GET /api/dashboard
```

### Description

Returns dashboard statistics including device summary, health metrics, and recent activity.

### Response

- 200 OK
- 500 Internal Server Error

---

# 4. Device APIs

## Get All Devices

### Endpoint

```
GET /api/devices
```

### Description

Returns all registered smart home devices.

---

## Get Device By ID

### Endpoint

```
GET /api/devices/{id}
```

### Description

Returns information about a specific device.

---

## Create Device

### Endpoint

```
POST /api/devices
```

### Description

Registers a new smart home device.

---

## Update Device

### Endpoint

```
PUT /api/devices/{id}
```

### Description

Updates an existing device.

---

## Delete Device

### Endpoint

```
DELETE /api/devices/{id}
```

### Description

Deletes a registered device.

---

## Device Heartbeat

### Endpoint

```
POST /api/devices/heartbeat
```

### Description

Receives heartbeat information from an ESP32 device.

The heartbeat updates:

- Device Status
- Last Seen
- Wi-Fi Strength
- IP Address

---

# 5. Activity APIs

## Get Activities

### Endpoint

```
GET /api/activities
```

### Description

Returns recent activity generated by connected devices.

---

## Record Motion Event

### Endpoint

```
POST /api/activities/motion
```

### Description

Receives motion detection events from ESP32 devices.

---

# 6. Sensor APIs

## Submit Sensor Reading

### Endpoint

```
POST /api/sensors/readings
```

### Description

Stores environmental sensor readings.

Collected data includes:

- Temperature
- Humidity
- Battery
- Signal Strength

---

## Get Sensor Readings

### Endpoint

```
GET /api/sensors/readings
```

### Description

Returns historical sensor readings.

---

# 7. Health APIs

## Get Health Metrics

### Endpoint

```
GET /api/health
```

### Description

Returns backend application health metrics.

Metrics include:

- Memory Usage
- CPU Usage
- Database Status

# 8. HTTP Status Codes

The backend uses standard HTTP status codes.

| Code | Description             |
| ---- | ----------------------- |
| 200  | Success                 |
| 201  | Resource Created        |
| 204  | Resource Deleted        |
| 400  | Bad Request             |
| 401  | Unauthorized _(Future)_ |
| 403  | Forbidden _(Future)_    |
| 404  | Resource Not Found      |
| 409  | Conflict                |
| 500  | Internal Server Error   |

---

# 9. Error Response Format

The backend returns standardized error responses.

Example:

```json
{
	"timestamp": "...",
	"status": 404,
	"error": "Not Found",
	"message": "Device not found",
	"path": "/api/devices/1"
}
```

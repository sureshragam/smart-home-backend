# Glossary

This glossary defines common terms, abbreviations, and concepts used throughout the Smart Home IoT Platform documentation.

---

# A

## Activity

A recorded event generated by the platform, such as motion detection, device status changes, or future automation events.

---

## API (Application Programming Interface)

A collection of HTTP endpoints that enables communication between the frontend, backend, and IoT devices.

---

## Authentication

The process of verifying the identity of a user before granting access to protected resources.

---

## Authorization

The process of determining whether an authenticated user has permission to access a resource or perform an action.

---

# B

## Backend

The Spring Boot application responsible for processing business logic, exposing REST APIs, managing devices, storing data, and coordinating communication between all platform components.

---

## BaseEntity

A common superclass inherited by all database entities that provides audit fields such as `createdAt` and `updatedAt`.

---

# C

## Controller

The REST layer responsible for receiving HTTP requests, validating input, and delegating processing to the service layer.

---

## CRUD

The four fundamental database operations:

- Create
- Read
- Update
- Delete

---

# D

## Dashboard

The main web interface that provides an overview of connected devices, health metrics, activities, and project statistics.

---

## Device

Any IoT hardware registered with the Smart Home Platform.

Examples include:

- ESP32
- ESP32-CAM
- Motion Sensors
- Relay Controllers

---

## Device Heartbeat

A periodic message sent from an IoT device to the backend indicating that the device is online and operational.

---

## Device Status

Represents the current availability of a device.

Typical values include:

- ONLINE
- OFFLINE

---

## DTO (Data Transfer Object)

An object used to transfer data between the backend and clients without exposing database entities directly.

---

# E

## Entity

A Java class mapped to a database table using JPA annotations.

---

## ESP32

A Wi-Fi and Bluetooth-enabled microcontroller used as the primary hardware platform for smart home devices.

---

## ESP32-CAM

An ESP32 development board equipped with a built-in camera module capable of image capture and video streaming.

---

# F

## Frontend

The React-based web application used to monitor and manage smart home devices through a graphical user interface.

---

# G

## GitHub Project

A project management board used to organize sprint planning, issues, priorities, and development progress.

---

# H

## Health Monitoring

Monitoring system resources such as CPU usage, memory consumption, application health, and database availability.

---

## Heartbeat

A scheduled communication from an IoT device confirming that it is connected and functioning correctly.

---

## Hibernate

The Object-Relational Mapping (ORM) framework used by Spring Data JPA to map Java entities to MySQL tables.

---

# I

## Infrastructure

The cloud and networking components that host and expose the platform, including AWS EC2, Nginx, and Apache Tomcat.

---

## IoT (Internet of Things)

A network of interconnected physical devices capable of collecting, transmitting, and receiving data over a network.

---

## Investment

A project-related purchase such as hardware, infrastructure, networking equipment, or software subscriptions tracked by the Project Investment module.

---

# J

## JPA (Java Persistence API)

The Java specification used for object-relational mapping and database persistence.

---

## JWT (JSON Web Token)

A secure token-based authentication mechanism used to authenticate and authorize users.

---

# L

## Layered Architecture

A software architecture pattern that separates responsibilities into distinct layers such as Controller, Service, Repository, and Database.

---

# M

## Mapper

A component responsible for converting between Entities and DTOs.

---

## Module

An independent functional area of the application, such as Device, Dashboard, Health, Sensor, or Investment.

---

## Motion Event

An event generated when a PIR sensor detects movement and reports it to the backend.

---

## MySQL

The relational database management system used for storing application data.

---

# N

## Nginx

A reverse proxy server used to securely expose the backend application over HTTPS.

---

# P

## Platform

The complete Smart Home IoT ecosystem consisting of the frontend, backend, database, infrastructure, and IoT devices.

---

## PIR Sensor

A Passive Infrared (PIR) sensor used to detect motion by measuring changes in infrared radiation.

---

## Project Investment

A module used to record and manage all expenses related to building and maintaining the Smart Home IoT Platform.

---

# R

## React

The frontend JavaScript library used to build the Smart Home Dashboard.

---

## Relay Module

An electronic switching component used to control high-voltage electrical appliances using an ESP32.

---

## Repository

The data access layer responsible for communicating with the database using Spring Data JPA.

---

## REST API

An HTTP-based interface that enables communication between software components using methods such as GET, POST, PUT, and DELETE.

---

# S

## Scheduler

A background process responsible for executing automated tasks at predefined intervals.

---

## Sensor

A hardware component that collects environmental or physical measurements such as temperature, humidity, motion, or light.

---

## Service Layer

The business logic layer responsible for processing requests, applying validation, and coordinating application workflows.

---

## Sprint

A fixed development cycle during which a defined set of features or improvements is implemented following Agile development practices.

---

## Spring Boot

The backend framework used to build RESTful services for the Smart Home IoT Platform.

---

# T

## Tomcat

The application server used to deploy and run the Spring Boot application as a WAR file.

---

# V

## Validation

The process of ensuring incoming requests satisfy predefined business and data integrity rules before processing.

---

# W

## WebSocket

A full-duplex communication protocol enabling real-time communication between the backend and frontend.

---

# Purpose

This glossary serves as a central reference for terminology used throughout the Smart Home IoT Platform documentation, ensuring consistent language across all technical documents and providing a common understanding of architectural, technical, and domain-specific concepts.
